UX review checklist

✓ Primary action prominent — "Validate & Export" is the only heavy button.
✓ Hierarchy — size → scale → bed → format → export, top to bottom.
✓ States shown — default, oversized-warning, validating/repairing, success, refused.
✓ Cancel / back — panel close (✕) everywhere; refused state routes back to sculpting.
✓ Destructive vs constructive — none destructive here; refusal is protective, framed helpfully not as an error.
△ Oversized is deliberately non-blocking (confirmed with owner) — copy must make "export anyway" feel intentional, not like ignoring a problem.

Annotations

  1. 1Live dimensions in mm (FR-10, BR-01) — always visible, updates with scale. This is also shown in the workspace status bar; the panel repeats it because size is the decision being made here.
  2. 2Uniform scale — one control, aspect-locked (a beginner scaling a figurine wants "bigger/smaller," not per-axis distortion). Slider + exact numeric entry; showing the resulting longest-side mm makes it concrete.
  3. 3Printer bed context (FR-11) — presets for common hobby printers + Custom. The top-down preview shows the model footprint against the plate so scale is judged at a glance.
  4. 4Format — STL default (MUST); 3MF second (SHOULD, FR-20). If 3MF is cut from a build, this control collapses to STL-only with no empty affordance.
  5. 5"Validate & Export" — the label names the guarantee. Validation always precedes writing; the button is the gate, not a plain "save."
  6. 6Oversized-for-bed = warning (owner-confirmed) — never blocks. Copy acknowledges legitimate reasons (print in parts, bigger printer) and the CTA becomes "Export anyway" so the choice is explicit.
  7. 7Validate → repair pipeline (BR-02, NFR-06) — visible, plain-language steps run in a Web Worker (NFR-03) so the UI never freezes. Auto-repair is attempted silently and reported after the fact.
  8. 8Success — confirms watertight, names the file, reports what was auto-repaired (trust through transparency), then offers Save (native handle or download per Flow 6).
  9. 9Refused (the core promise made visible) — when repair can't guarantee watertightness, zmesh refuses and explains in plain language, pointing at the problem area in the viewport and offering concrete fixes. Never a stack trace, never a silently-broken file.
