UX review checklist

✓ Primary action prominent — "+ New sculpt" in the header and as the first grid tile.
✓ Hierarchy — create is first; recent projects ordered by last edited.
✓ States shown — populated, shape-picker, rename, delete-confirm, empty.
✓ Destructive clearly different — Delete is red, requires a confirm dialog, and reassures about the on-disk copy.
✓ Way back — from a project the workspace has "← Gallery"; the gallery is the top of the tree.
△ Thumbnails depend on autosave capturing a viewport snapshot — placeholder shown until first save.

Annotations

  1. 1New sculpt (FR-12) — offered twice (header button + leading dashed grid tile) so it's unmissable. Opens the shape picker.
  2. 2Starting-shape picker (FR-22, SHOULD) — sphere pre-selected so the MUST behaviour (straight to a sphere) is one Enter-press away. Egg/Block/Capsule cover the common figurine / functional-print starts. If FR-22 is cut, "+ New sculpt" just creates a sphere with no dialog.
  3. 3Project card — thumbnail + name + last-edited (and size when known). Click opens; the ⋮ kebab reveals per-project actions.
  4. 4(context menu) — Open / Rename / Duplicate (FR-32, COULD) / Delete. Duplicate is shown greyed-in for placement but is a COULD; safe to omit from v1.
  5. 5Open .zmesh… (FR-14) — loads a project file from disk via the native picker (Chrome/Edge) or an upload fallback (Safari/Firefox). Invalid files route to the error state (Flow 8, separate wireframe).
  6. 6Local-first reassurance (NFR-07) — a quiet line stating data stays in the browser; sets expectations honestly (also implies: clearing browser data removes projects — worth surfacing near disk-save).
  7. 7Rename (FR-12) — lightweight modal; also reachable by clicking the name in the workspace top bar.
  8. 8Delete confirmation — destructive and irreversible from the UI, so it requires explicit confirm, uses the only red button, and clarifies that a disk .zmesh copy is not affected (BR-05 mental model).
  9. 9Empty state — rarely seen because first run auto-creates a sphere project (Flow 1); it appears only if a user deletes everything. Encouraging, single clear CTA.

Low-fidelity — grayscale, structural. Sample project names/dates are illustrative.
