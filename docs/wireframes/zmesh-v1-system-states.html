System States

The cross-cutting states that guard the edges of the experience: an unsupported browser (Flow 1), recovering unsaved work after a crash (Flow 7), and a bad project file (Flow 8). All three share one principle — fail safe and explain in plain language; never crash, never lose data silently.

UX review checklist

✓ Fails safe — every state preserves current work; nothing is destroyed without confirmation.
✓ Plain language — no stack traces, no error codes; each message says what happened and what to do.
✓ Destructive clearly different — recovery "Discard" is red and paired with a safe default (Restore).
✓ Blame-free tone — "It's not you"; problems framed as the app's job to handle.
△ Capability notice is terminal (no in-app path forward) — must be genuinely helpful since it's a dead end.
△ "Discard" is the only irreversible action here — confirm copy is unambiguous before build.

Annotations

  1. 1Unsupported-browser notice (Flow 1, NFR-05) — shown only when neither WebGPU nor WebGL2 is available, before any canvas init. It's a terminal state, so it must be maximally helpful: name the supported browsers, reassure it's a capability gap, avoid jargon. No broken viewport is ever shown.
  2. 2Recovery prompt (Flow 7, FR-13) — appears at open when IndexedDB holds autosaved changes newer than the last committed project state. Shows a thumbnail + how much work is at stake so the choice is informed. Restore is the safe default (primary).
  3. 3Discard — the one destructive control in these states; red, non-default. Choosing it keeps the last saved state (never deletes the project itself). Recovery restores mesh state only — the undo stack is not persisted (BR-04).
  4. 4Invalid/corrupt file (Flow 8, NFR-08) — imported files are untrusted; a defensive parser rejects bad input as a non-blocking toast. Crucially: current work is untouched — a bad load never mutates session state.
  5. 5Version mismatch — a valid but newer-format file gets its own message (distinct from "corrupt") offering the real fix: refresh to update the app. Forward-compat honesty for a local-first app that updates by reload.

Low-fidelity — grayscale, structural. Copy is indicative and will be refined during implementation.
