UX review checklist

✓ Primary action prominent — Export is the only heavy-bordered button, top-right.
✓ Hierarchy clear — pick brush (rail) → tune size/strength → sculpt in viewport → export.
✓ Alternate states shown — remesh-in-progress and detail-ceiling clamp.
✓ All controls labeled — brushes named + numbered; sliders show live values.
✓ Way back — "← Gallery" is the persistent up-navigation; autosave means it is non-destructive.
✓ Destructive vs. constructive — no destructive action lives on this screen (delete is in the gallery).
△ Empty state N/A — the workspace always has a mesh (opens on a sphere); there is no empty viewport by design (FR-01).
△ Discoverability risk — camera-vs-brush model must be teachable; mitigated by the always-available "?" legend (see annotation 8). Confirm during usability.

Annotations

  1. 1Back to Gallery — persistent up-navigation. Safe to click any time; the session is autosaved to IndexedDB (FR-13), so no "unsaved changes" nag is needed for the autosave copy. (A separate prompt still applies to the explicit on-disk .zmesh if desired.)
  2. 2Undo / Redo — ≥50 steps per session (FR-07). Every brush stroke and every remesh is one atomic step. Shortcuts: Cmd/Ctrl+Z / Shift+Cmd/Ctrl+Z.
  3. 3Detail slider — the single "how fine" control (FR-08). Discrete labelled stops (Low / Med / High / Max) rather than raw triangle counts — beginners think in "smoothness," not topology. Changing it triggers the remesh in state 2.
  4. 4Mirror X toggle — ON by default (FR-06), shown active. The dotted vertical line in the viewport is the mirror plane. Prominent because symmetry is the biggest quality win for figurines/faces.
  5. 5Export — the primary action and the product's core promise. Opens the Export panel (separate wireframe) where scale/bed/format/validation live.
  6. 6Brush rail — all seven brushes visible at once (FR-03), no nested menus. Number-key shortcuts 1–7. Active brush is heavier-bordered + filled.
  7. 7Contextual brush parameters — Size + Strength for the active brush (FR-05), with the invert-modifier hint inline (FR-04). Panel header names the active brush so the context is unambiguous.
  8. 8Camera-vs-brush input model (THE key decision — settled). One unifying rule per device: act on the model with the "primary" input, move the camera with the "secondary" one. Surfaced via the always-available "?" legend.
    Trackpad (primary target — MacBook): the rule is one finger = sculpt/act, two fingers = camera.
    • 1-finger drag on the model → sculpt · on empty → orbit
    • 2-finger drag → orbit (always works), even when the model fills the viewport
    • Pinch → zoom · Shift+2-finger → pan · Option → invert brush
    Mouse: left-drag on model → sculpt · left-drag on empty → orbit · right-drag → orbit anywhere · scroll → zoom · middle/Shift+right → pan · Alt → invert.
    This mirrors the Nomad Sculpt (iPad) convention, is fully one-handed on a laptop trackpad, and needs no right-button or middle-button. The "always works" secondary-orbit (2-finger / right-drag) is the escape hatch for re-framing when zoomed in — it replaces the earlier Hold-Space idea, now dropped. The dashed circular brush cursor only appears when hovering the model, giving a constant "sculpt-mode-here" cue.
  9. 9Remesh progress overlay — detail changes run in a Web Worker (NFR-03, ≤3 s at max); a lightweight non-blocking overlay communicates work without freezing the app. Undoable when done.
  10. 10Detail ceiling clamp — the slider physically stops at Max (BR-03, budget set by the Q-01 60 fps benchmark). Plain-language toast explains why and offers alternatives; it is never framed as an error.

Low-fidelity wireframe — grayscale and structural only. Icons are placeholder text labels; final iconography, exact spacing, and theming are out of scope here.
